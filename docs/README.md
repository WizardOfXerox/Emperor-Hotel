# Emperor Hotel Documentation

This folder contains the main project documentation for the Emperor Hotel Reservation and Management System.

## Main Documentation Files

| Document | Purpose |
| --- | --- |
| `system-boundaries.md` | Explains what the system will do, what it will not do, the major components, and the deployment environment. |
| `database-usage.md` | Shows database creation, table examples, insert/update/delete/query examples, and the ERD model. |
| `website-flowchart.md` | Shows the website flow for guests, registered users, and administrators. |
| `definition-of-terms.md` | Defines technical terms used in the project, such as CRUD, API, Bootstrap, ERD, PDO, and XML. |
| `technology-stack.md` | Lists the actual technologies and local browser libraries used by the project. |
| `database-erd.md` | Provides a detailed database ERD and table relationship explanation. |
| `erd-file-correlation.md` | Maps each ERD table to the PHP models, pages, includes, and UI files that use it. |
| `dashboard-feature-tracker.md` | Tracks implemented, partial, missing, and future dashboard/admin features. |
| `project-explanation-guide.md` | Explains how the whole project works for presentation, defense, and code walkthroughs. |
| `code-explanation.md` | Gives a detailed code-level walkthrough of folders, models, pages, reservation flow, payments, room XML, validation, and the Manage modal. |

## Important Note

This project is a Core PHP and MySQL hotel reservation and management system. Reservations are the core workflow, while room, guest, payment, check-in/check-out, receipt, and report modules support the full hotel process. It does not use React.js, Tailwind CSS, PostgreSQL, Laravel, Node.js, or a mobile app framework.
